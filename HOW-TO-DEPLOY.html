ขั้นตอนที่ 1 — Database (ฟรี · 10 นาที)

ตั้งค่าSupabase

Supabase = Database (PostgreSQL) + Auth (Login) + Storage (รูปภาพ) ฟรีทั้งหมด — ไม่ต้องเขียน Backend

1
สมัคร Supabase (ฟรี)
ฟรี 100% ⏱ 3 นาที ใช้ Google account สมัครได้เลย
›
1
เปิด supabase.com
ไปที่ supabase.com → คลิก "Start your project" → Sign up ด้วย GitHub หรือ Google (แนะนำ GitHub เพราะใช้ร่วมกันได้)
2
สร้าง Organization และ Project
คลิก "New project" → กรอก:
• Organization: ชื่อบริษัท/ทีมของคุณ
• Name: thailand-market
• Database Password: สร้างรหัสผ่านที่แข็งแกร่ง (จด password ไว้!)
• Region: Southeast Asia (Singapore) ← สำคัญมาก เพื่อความเร็วในไทย
• Plan: Free
→ คลิก "Create new project" รอ ~2 นาที
3
คัดลอก API Keys
ไปที่ Project Settings (ไอคอน ⚙️ ซ้ายล่าง) → API → คัดลอก 2 ค่านี้:
Project URL: https://xxxxxxxxxxxxxxxxxxxx.supabase.co anon public key: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOi... # ⚠️ อย่า copy "service_role" key ใส่ในไฟล์ HTML ที่ user เห็นได้!
2
รัน SQL เพื่อสร้าง Database
สำคัญมาก ⏱ 2 นาที ทำครั้งเดียว สร้าง Tables + ข้อมูลเริ่มต้นทั้งหมด
›
1
เปิด SQL Editor
ใน Supabase Dashboard → คลิก "SQL Editor" (ไอคอน </> ในแถบซ้าย) → คลิก "New query"
2
Copy SQL แล้ว Paste และ Run
เปิดไฟล์ supabase-setup.sql ในโฟลเดอร์ → Select All → Copy → Paste ใน SQL Editor → คลิก "Run" (สีเขียว)

จะสร้าง: ตาราง users, categories, products, Storage bucket, RLS policies + ข้อมูลเริ่มต้น 10 หมวดหมู่
3
ตรวจสอบว่า Run สำเร็จ
ไปที่ Table Editor → ควรเห็น tables: users, categories, products
ไปที่ Storage → ควรเห็น bucket: product-images
⚠️ ถ้าเห็น Error "already exists" ไม่เป็นไร — หมายความว่า table มีอยู่แล้ว ผลลัพธ์ยังถูกต้อง ไม่ต้องทำอะไรเพิ่ม
3
เปิด Authentication + สร้าง Admin User
⏱ 3 นาที สร้างบัญชี Admin คนแรกสำหรับ Login CMS
›
1
เปิด Email Auth
ไปที่ Authentication → Providers → Email → Toggle Enable → Save
2
สร้าง Admin User คนแรก
ไปที่ Authentication → Users → "Invite user" หรือ "Add user"
→ กรอก Email และ Password ของ Admin
→ คลิก "Create user"
3
Set role = admin ใน Database
ไปที่ SQL Editor → New query → รัน SQL นี้ (แทนที่ email ด้วยของคุณ):
-- แทนที่ email และ display_name ด้วยค่าของคุณ INSERT INTO users (email, display_name, role, is_active) VALUES ('admin@yoursite.com', 'Admin', 'admin', true) ON CONFLICT (email) DO UPDATE SET role = 'admin';
4
(ทางเลือก) เชิญ Seller
Authentication → Users → Invite user → กรอก Email Seller → Role จะเป็น 'seller' โดย Admin กำหนดใน CMS ได้
4
ตั้งค่าเว็บด้วย setup.html
⏱ 1 นาที กรอก URL + Key ครั้งเดียว บันทึกในเบราว์เซอร์
›
1
เปิดไฟล์ setup.html
เปิดไฟล์ setup.html ในเบราว์เซอร์ (double-click หรือเปิดผ่าน Netlify URL หลัง deploy)
2
กรอก Supabase URL และ Anon Key
วาง Project URL และ anon public key ที่คัดลอกจาก Supabase → คลิก "บันทึกและเริ่มใช้งาน"
3
Login CMS ทดสอบ
จะ redirect ไป admin-cms.html → กรอก Email/Password ที่สร้างใน Supabase → Login → ลองเพิ่มสินค้าทดสอบ → เปิด storefront-live.html เพื่อดูสินค้าที่แสดงบนเว็บ
💡 ทำไม URL + Key ถึงปลอดภัย? Supabase "anon key" เป็น key สาธารณะที่ออกแบบมาให้ใช้ใน frontend ได้ · Row Level Security (RLS) ใน Database ควบคุมว่าใครเห็นและแก้ไขอะไรได้ · ไม่มีใครสามารถลบหรือแก้ข้อมูลคนอื่นได้ผ่าน key นี้

วิธีที่ 1: Netlify Drop (ง่ายสุด · 2 นาที)
ฟรี ⏱ 2 นาที แนะนำ
›
1
Zip ไฟล์ทั้งหมด
Windows: เลือกทุกไฟล์ในโฟลเดอร์ → Right-click → "Compress to ZIP file" (หรือ Send to → Compressed folder)
Mac: เลือกทุกไฟล์ → Right-click → "Compress X items"

⚠️ Zip เนื้อหาข้างใน ไม่ใช่โฟลเดอร์ทั้งก้อน — เปิด zip แล้วควรเห็น index.html ทันที ไม่ใช่โฟลเดอร์อีกชั้น
2
ไปที่ app.netlify.com/drop
ไม่ต้องสมัครก็ได้ → Drag ZIP ลงในพื้นที่ที่ระบุ → รอ 10–30 วินาที → ได้ URL แบบ random-name-abc123.netlify.app
3
(ทางเลือก) สมัครเพื่อจัดการได้
Sign up ฟรี → Claim site → ตั้งชื่อ subdomain เป็น thailand-market.netlify.app → หรือเชื่อม Custom domain ของตัวเอง (เช่น myshop.co.th)
4
เปิด setup.html บน URL ใหม่
เปิด https://your-site.netlify.app/setup.html → กรอก Supabase URL + Key → บันทึก → เริ่มใช้งานได้เลย!
2
วิธีที่ 2: GitHub Pages (ถาวร · 5 นาที)
ฟรี ⏱ 5 นาที เหมาะถ้าต้องการ update ไฟล์บ่อย
›
1
สมัคร GitHub (ถ้ายังไม่มี)
ไปที่ github.com → Sign up → ยืนยัน email → เสร็จ
2
สร้าง Repository ใหม่
คลิก "New" (สีเขียว) → Repository name: thailand-market → เลือก Public → คลิก "Create repository"
3
Upload ไฟล์ทั้งหมด
ใน Repository ที่สร้าง → คลิก "uploading an existing file" → Drag ไฟล์ทั้งหมดมาวาง (ทีละ 50 ไฟล์ หรือ drag โฟลเดอร์) → กรอก Commit message: "Initial deploy" → คลิก "Commit changes"
4
เปิด GitHub Pages
Settings → Pages → Source: Deploy from a branch → Branch: main → Folder: / (root) → Save → รอ 2–3 นาที → URL: https://username.github.io/thailand-market/
3
เชื่อม Custom Domain (ทางเลือก)
⏱ 15 นาที + รอ DNS 24 ชม. เช่น myshop.co.th — ราคา domain ประมาณ ฿500/ปี
›
1
ซื้อ Domain
ไทย (.co.th / .th): thnic.co.th หรือ GoDaddy
ทั่วไป (.com / .shop): Namecheap (ถูกสุด) หรือ Google Domains
ราคา: .com ≈ ฿400/ปี · .co.th ≈ ฿500/ปี · .shop ≈ ฿200–500/ปี
2
ตั้งค่า DNS ที่ Registrar
ไปที่ DNS Settings ของผู้ขาย domain → เพิ่ม record:
# สำหรับ Netlify: Type: CNAME Name: www Value: your-site.netlify.app TTL: Auto # สำหรับ root domain (@): Type: A Name: @ Value: 75.2.60.5 (Netlify IP)
3
เพิ่ม Domain ใน Netlify/Vercel
Netlify: Site settings → Domain management → Add custom domain → กรอก domain → Verify
Vercel: Project → Settings → Domains → Add → กรอก domain

SSL Certificate (HTTPS) จะสร้างอัตโนมัติ ฟรี ใช้เวลา 5–30 นาที
💡 DNS ใช้เวลา propagate 1–48 ชั่วโมง ปกติ 1–2 ชั่วโมงก็ใช้ได้แล้วในไทย · ระหว่างรอยังเข้าผ่าน URL ของ Netlify/Vercel ได้ปกติ
